Infinite Canvas Agent App

Lista de funcionalidades com a premissa correta: cada workspace é uma pasta ou repositório local aberto no app, e sua configuração visual/operacional é salva dentro do próprio projeto.

Definição base

Um workspace não é uma empresa, cliente ou espaço SaaS. Ele representa uma raiz local de trabalho: uma pasta, projeto ou repositório Git.

C:/Projetos/meu-app D:/Repos/minha-api C:/Clientes/site-institucional

Persistência

O app cria uma pasta oculta dentro do projeto para salvar canvas, nodes, edges, sessões, regras, contexto e configurações.

meu-app/ .agentcanvas/ workspace.json canvas.json nodes.json edges.json sessions/ logs/ memory/ workflows/
Local-first

O projeto deve funcionar localmente antes de depender de servidor.

Canvas como core

O canvas é o espaço visual de trabalho, não apenas decoração.

Go como executor

Go executa terminal, agentes, Git, arquivos e processos.

Elixir como control plane

Elixir coordena estado, eventos e workers, sem executar tudo diretamente.
